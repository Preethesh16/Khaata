1. THE PROBLEM STATEMENT
--------------------------------------------------------------------------------
India has 65+ million kirana shops (small family-run grocery stores). Their
billing reality:
  - Paper notebooks ("khaata" literally means account ledger in Hindi)
  - The shopkeeper's hands are never free — serving 5 customers at once
  - Many speak Hindi/Kannada, not English; many are not smartphone-literate
  - Internet in tier-2 market lanes is patchy or absent
  - Existing POS apps require tapping through screens — nobody uses them
  - Billing data is cash-business-sensitive; owners don't want it in a cloud

KHAATA is the answer: a local-first Android VOICE agent. The shopkeeper just
TALKS — "do kilo cheeni, ek Parle-G, teen Maggi" — and the app builds an
itemized bill, decrements stock, and speaks confirmation back in Hindi.
It works FULLY OFFLINE when connectivity drops. A camera scan can also add
products. If a 60-year-old shopkeeper can't use it in 10 seconds, we redesign.


2. THE DIFFERENTIATOR (why this wins)
--------------------------------------------------------------------------------
Every other team picks ONE Google AI tool and builds around it. Khaata chains
the Google stack into ONE coherent pipeline:

  DUAL-MODE VOICE BRAIN:
    ONLINE  -> Gemini Live API   (real-time voice, function calling)
    OFFLINE -> Gemma 3n on-device (same function calls, zero internet)
  The app watches connectivity and switches brains AUTOMATICALLY.
  The shopkeeper never notices the switch. Airplane mode IS the demo:
  we put the phone in airplane mode in front of the judges and it still bills.

  Other differentiators:
  - No backend server AT ALL. Firebase AI Logic SDK calls Gemini directly
    from the app. Two students built and can deploy the whole thing.
  - Privacy by construction: offline mode = data never leaves the device.
  - Production-real: Apache-2.0 Gemma, free Firebase tier, Room DB.
    Could pilot in 100 Mangalore shops tomorrow.
  - The demo is 30 seconds and needs no explanation.


3. GOOGLE TOOLS USED (all five, chained, not a checklist)
--------------------------------------------------------------------------------
1. GEMINI LIVE API (model "gemini-live-2.5-flash-preview")
   - Online voice mode. Full-duplex audio streaming: mic audio up, model's
     spoken audio back. Native barge-in. 70+ languages, handles Hinglish.
   - Called via the Firebase AI Logic Android SDK ("firebase-ai" artifact):
     Firebase.ai(backend = GenerativeBackend.googleAI()).liveModel(...)
     session.startAudioConversation(functionCallHandler)
   - The model calls OUR 7 billing tools (FunctionDeclarations) directly.

2. GEMMA 3n ON-DEVICE (E4B int4 = 4.41 GB, E2B int4 = 3.14 GB)
   - RAM RULE (learned on a real device): E4B needs a flagship-RAM phone
     (>= 10 GB free); on a 6-8 GB phone Android kills the app with
     LOW_MEMORY during load — use E2B there. Check with:
     adb shell cat /proc/meminfo | head -1
   - Both load from the SAME path; zero code change to swap.
   - Offline brain. Runs via MediaPipe LLM Inference / LiteRT
     (dependency com.google.mediapipe:tasks-genai).
   - Parses the shopkeeper's utterance into structured ITEM|QTY lines,
     which drive the SAME 7 tools as online mode.
   - Weights load from /data/local/tmp/llm/gemma.task (adb-pushed).
   - SAFETY NET: if weights are missing/slow, a deterministic rule-based
     parser (regex + Hindi number words + fuzzy catalog match) takes over.
     The offline demo can NEVER die.

3. FIREBASE AI LOGIC (the platform glue)
   - Project "khaata-kirana", Android app com.khaata.app registered.
   - Enabled APIs: generativelanguage.googleapis.com (Gemini Developer API),

4. OMNI FLASH / GEMINI FLASH VISION (model "gemini-2.5-flash")
   - The wildcard: point the camera at a product packet -> CameraX captures
     a frame -> Gemini vision returns strict JSON
     {productName, quantity, unit, confidence}
   - Confidence gates: >=0.7 auto-add to bill; 0.4-0.7 ask "Yeh X hai?";
     <0.4 say "couldn't identify, please speak the item".
   - Feeds the SAME tool pipeline as voice. One brain, three input modes.

5. HOW THE APP WORKS (user's point of view)
--------------------------------------------------------------------------------
 1. Launch. Catalog auto-seeds on first run. Badge shows ONLINE (green) or
    OFFLINE (orange). Bill area says "Bolo apna order".
 2. Tap the big saffron mic. Speak: "do kilo cheeni aur ek Parle-G".
 3. ONLINE: Gemini Live hears the audio stream, calls lookup_price ->
    add_to_bill -> update_stock per item, and SPEAKS confirmation back.
    OFFLINE: on-device STT transcribes -> Gemma (or rule parser) extracts
    items -> same tools -> Android TTS speaks "Cheeni 2 — 80 rupaye".
 4. Each item slides into the bill list; TOTAL updates; stock ticks down;
    red warning strip appears if any stock < 3.
 5. Corrections: say "hatao"/"remove" -> last item removed, stock restored.
    "khatam"/"bas"/"total" -> spoken summary. "nayi bill" -> fresh customer.
 6. Or tap SCAN: point camera at a packet -> Gemini vision identifies it ->
    auto-adds (or asks "Yeh Maggi hai?" if unsure).
 7. Tap DONE: summary screen — itemized bill, total, item count, SHARE as
    text (WhatsApp-ready), NAYI BILL to start over.
 8. ↺ button (top bar) = demo reset: empty bill + full 50-item stock.


6. HOW TO BUILD & INSTALL ON A PHONE VIA USB (step by step)
--------------------------------------------------------------------------------
 PREREQUISITES on the computer:
   - JDK 17, Android SDK (Linux box: /home/infinity/Android/Sdk;
     Deepthi's Windows/ARM64 laptop: C:\tools\jdk17 + SDK, build with
     cmd.exe: set JAVA_HOME=C:\tools\jdk17 && gradlew.bat assembleDebug)
   - local.properties containing: sdk.dir=<path-to-Android-SDK>
   - app/google-services.json in place (see section 3.3) for online mode.

 STEP 1 — enable Developer Mode on the phone:
   Settings -> About phone -> tap "Build number" 7 times
   -> Settings -> Developer options -> enable "USB debugging"

 STEP 2 — connect phone via USB cable, then on the computer:
   cd ~/Projects/Khaata
   ~/Android/Sdk/platform-tools/adb devices
   # phone shows a popup "Allow USB debugging?" -> tick Always allow -> OK
   # re-run adb devices: it must say "<serial>  device" (not "unauthorized")

 STEP 3 — build the APK:
   ./gradlew assembleDebug
   # output: app/build/outputs/apk/debug/app-debug.apk (~64 MB)

 STEP 4 — install:
   ~/Android/Sdk/platform-tools/adb install -r app/build/outputs/apk/debug/app-debug.apk
   # "-r" replaces an existing install, keeping data

 STEP 5 — push the Gemma offline model (one-time, phone needs ~4-5 GB free):
   # FIRST check phone RAM: adb shell cat /proc/meminfo | head -1
   #   >= 10 GB -> E4B weights ok; 6-8 GB -> use E2B weights (3.14 GB)
   ~/Android/Sdk/platform-tools/adb shell mkdir -p /data/local/tmp/llm
   ~/Android/Sdk/platform-tools/adb push models/gemma.task /data/local/tmp/llm/gemma.task
   # Takes several minutes over USB. Survives app reinstalls.
   # Without it, offline mode still works via the rule-based parser.

 STEP 6 — first-run prep on the phone (IMPORTANT, do this EARLY on wifi):
   - Open Khaata, grant Microphone + Camera permissions.
   - Tap the mic once while on wifi: if the Hindi offline speech pack is
     missing, the app now self-triggers its download (Deepthi's fix for
     LANGUAGE_PACK_ERROR 13; newer Google app removed the manual menu).
     Leave the phone plugged in on wifi for a while so the pack lands.
     Until it does, the app auto-falls back to en-IN recognition
     (Hinglish still parses fine).
   - Verify: airplane mode ON -> badge flips OFFLINE -> speak an order.

 TROUBLESHOOTING:
   - adb shows "unauthorized" -> unlock phone, accept the popup.
   - adb shows nothing -> try another cable/port; check USB mode is not
     "charge only" (pick File transfer); adb kill-server && adb devices.
   - Mic does nothing offline -> speech pack still downloading; wait,
     or demo in en-IN.
   - DEMO DAY RULE: pre-install the APK; do NOT rely on live ADB during
     judging.

 JUDGE Q&A AMMO:
 - Why not a cloud chatbot? Venue wifi isn't guaranteed in a market lane,
   and cash businesses don't want billing data leaving the device.
 - Why not existing POS apps? They need free hands and literacy. Voice is
   the natural interface; it only became possible on-device with Gemma.
 - Dialects/code-mixing? Gemini Live: 70+ languages. Gemma: 140+. Fuzzy
   matcher + alias map handles mishears. Tested with Hinglish.
 - Path to production? Gemma is Apache 2.0, Firebase free tier, catalog is
   configurable per shop. Pilot 100 shops in Mangalore tomorrow.
